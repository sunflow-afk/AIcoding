# CLAUDE.md — AutoCare AI 项目规范

## 项目概述

**AutoCare AI** — 尼日利亚二手车维护助手，融合 AI 诊断、维修师匹配、车辆档案与保养提醒，帮助车主判断故障风险、减少乱报价和维修不确定性，同时统一市场。

- **项目性质**：公司内部创新大赛（新入职员工参赛）
- **目标市场**：尼日利亚（Lagos、Abuja 为核心城市）
- **战略依托**：传音手机（Tecno/Infinix/Itel）系统预装 + Palm Store 分发
- **目标用户**：网约车/出租车司机、普通二手车车主、小商户车主
- **核心差异化**：AI 诊断 + 技师匹配 + 车辆档案 + 保养提醒，四位一体

## 项目约定

### 语言
- 文档正文：**简体中文**
- 专有名词：保留英文原词（Tokunbo、Danfo、NAIDP、agbero 等），首次出现标注中文解释
- 货币单位：尼日利亚奈拉 `₦`，重要数据加美元换算 `(~$X)`
- 日期格式：`YYYY-MM-DD`

### 数据引用
- 所有市场数据需标注来源（NBS、NPA、NCC、FRSC、Canalys 等）
- 数据时间跨度：**2010–2025**（近 15 年），优先引用最新数据
- 估算数据标注"估算"，正式数据标注具体来源
- 文档底部列出完整数据来源清单

### 编辑规范
- 表格对齐，使用 Markdown 表格语法
- 章节编号用中文数字（一、二、三…），子章节用阿拉伯数字（2.1, 2.2…）
- 重要结论用 `**加粗**` 突出
- 用户洞察用 `>` 引用块

### Git 约定
- 分支：`master`（主分支）
- 远程：`origin` → `https://github.com/sunflow-afk/AIcoding.git`
- Commit 语言：中文或中英混合
- Commit 格式：`feat/fix/docs: 简短描述`
- Co-Authored-By: Claude \<noreply@anthropic.com\>

## 常用任务工作流

### 更新市场数据
1. 搜索最新年份数据（优先 NBS、NCC、NPA 等官方来源）
2. 在 `AutoCareAI_前期分析.md` 对应表格中追加新行/列
3. 检查相关章节的文字结论是否需要同步更新
4. Commit: `docs: 更新 [年份] [数据类型] 数据`

### 添加新 Persona
1. 在第五章按照现有格式新增
2. 模板：年龄/所在地 → 家庭/教育/收入 → 手机/车辆 → 用车场景 → 核心需求 → 目标 → 挑战 → 一句用户洞察
3. 同步更新第四章"目标客户分类表"（如有新类别）
4. 同步更新第六章"旅程地图"（如需要新场景）

### 新增章节
1. 使用中文数字序号（十、十一…）
2. 保持现有表格风格和排版
3. 更新本 CLAUDE.md 的章节索引

### 修改产品方向
1. 先更新 `目标.txt`
2. 同步更新 `AutoCareAI_前期分析.md` 第一章（核心目标阐述）
3. 检查后续章节是否仍一致
